Linux指令总结

2019-10-22

1、解决切换jdk8之后IDE快捷方式打不开问题

现象

开发虚机使用的是ubuntu系统，切换jdk8之后，重启虚机，发现通过左侧快捷方式打开IDE如Idea、Webstorm、Pycharm，但都报”NO JDK foud.Please validate either IDEA_JDK, JDK_HOME or JAVA_HOME environment variable points to valid JDK installation.”错误。令人意外的是，直接在安装目录，执行运行脚本却正常。

解决办法，直接配置/etc/environment的环境变量

JAVA_HOME=/usr/local/zxjdk-8u181-linux-x64-b01

重启后生效

2、Syntax error: “(“ unexpected (expecting “fi”)

在Ubuntu 14上执行脚本时，总是报告某行“Syntax error: “(“ unexpected (expecting “fi”)”错误。查了一下脚本没有发现错误，经过搜索说可能是sh的链接问题。
首先到/bin/下面查看当前链接：ls -l /bin/sh，发现sh默认是指向dash的。
经过搜索发现dash和bash是不同的（一直把dash看成bash了！！），那么到底是bash还是该是dash呢，Ubuntu安装时默认使用dash，据说dash比bash体积小速度快，但原来在bash下正常运行sh脚本在dash下运行往往有问题，而且提示错误模糊，说明dash的兼容似乎也有问题。

不过已经有人发现规避问题的方法，就是重新配置dash：

sudo dpkg-reconfigure dash

会弹出重新配置dash界面，并选择“no”不使用dash。

再执行脚本，发现问题解决~
