网站安全防卫的一次实践（解决被挂马跳转至博彩页面问题）

前两天，阿姐找到我说公司官网被黑了，从各种搜索引擎点进去，都会自动跳转到博彩网站上去，但是直接输入网址却一切正常，有点不可思议。一开始以为是域名劫持，后来发现不是。从CSDN和知乎上，分别找到类似问题的帖子，大致猜到应该是有人篡改了代码。

现象

![phenomenon picture][phenomenon]

找到可疑代码

确定了大致思路，一切就好办了。通过Ftp登陆部署主机，检查网站的根目录，并没有发现陌生的可疑文件，但是发现index.php近期被修改过。

于是静下心来，打开index.php文件，仔仔细细的一行一行走查代码。抑或是出于程序员的本能，很快就定位了可疑代码，事实表明运气真不错。

</script>

至于为啥怀疑这段代码，是因为几乎所有的js程序员都知道，在js中不建议使用eval函数。因为eval的东西都不会再有别的防御机制去审视要执行的是什么, 比如您要eval一个表单里的某串text，这就足以让植入式病毒码 (injection code) 长驱直入, 成为您页面里的一部分。正是eval函数本身具有安全性问题，通常代码中很少见，这里出现自然成为最大的怀疑对象。
